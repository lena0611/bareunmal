# 범위 계약

## 현재 범위 상태
- 현재 저장소는 특정 업무 도메인과 스택을 강제하지 않는 공통 하네스 상태입니다.
- 적용 대상 프로젝트가 신규 구축인지, 기존 안정 프로젝트의 유지보수인지, 마이그레이션인지 먼저 확인해야 합니다.

## 지금 확정된 범위
- 세션 복구 하네스 유지
- 기준-코드 동기화 하네스 유지
- 문서/검증 기준 유지
- 스택 하네스와 프로젝트 로컬룰의 공존 지원

## 아직 확정되지 않은 범위
- 적용 대상 프로젝트 상태
- 운영 책임 범위
- 반복 유지보수 업무
- 도메인 모델
- 외부 시스템 계약
- 릴리스와 장애 대응 기준

## 범위 추가 규칙
- 새로운 범위는 먼저 `project-charter.md`에 반영합니다.
- 범위가 구조적 제약을 바꾸면 기준 문서도 같이 갱신합니다.
- 범위가 바뀌었는데 헌장이 비어 있으면 새 기능 구현보다 현재 프로젝트 상태와 책임 범위를 먼저 확인합니다.
