# 스타일 규칙

저장소 전체에서 스타일 규칙을 어디서 읽고 어떻게 검증할지 정하는 하네스입니다. 구체 스타일 값은 프로젝트 로컬 방법론, formatter/linter 설정, 적용된 스택 프리셋 로컬 규칙을 우선합니다.

## 스타일 출처 우선순위
1. 프로젝트 로컬 방법론과 기존 전용 하네스
2. 프로젝트의 formatter/linter 설정
3. `stack:apply`로 정착된 `.harness/project/stack-preset-rules.md`
4. 공통 하네스 스타일 기준

## 공통 원칙
- 세미콜론, quote, import 정렬 방식처럼 프로젝트마다 달라질 수 있는 값은 로컬 규칙을 따릅니다.
- 공통 하네스는 구체 스타일 값을 전역으로 강제하지 않습니다.
- 로컬 규칙이 없을 때만 적용된 스택 프리셋의 기본 scaffold 설정을 참고합니다.
- 파일 끝 개행, 생성물 제외, formatter/linter 실행 경로처럼 도구 운영에 가까운 항목은 공통 하네스에서 안내할 수 있습니다.

## 스타일과 구조 규칙의 우선순위
- 스타일 규칙은 구조 규칙보다 우선하지 않습니다. 아키텍처 계층 규칙을 깨면서 스타일을 맞추지 않습니다.
- 활성 스택이 추가하는 스타일은 `stack:apply` 이후 `.harness/project/stack-preset-rules.md`에 정착된 로컬 규칙을 따릅니다.

## 주석
- 필요한 경우에만 주석을 추가합니다.
- 주석은 한국어로 작성합니다.

## 주의
- 스타일 규칙은 현재 자동화 범위 내에서 검증되는 것부터 우선 적용합니다.
- 의미적 판단이 필요한 스타일은 lint보다 리뷰/정책 판단 영역으로 남을 수 있습니다.
